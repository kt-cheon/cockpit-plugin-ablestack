PowerFlex Manager Platform 가상머신 배포 마법사

현재 호스트에 PowerFlex Manager Platform 가상머신을 구성하기 위해 단계별로 실행합니다.

PowerFlex Manager Platform 가상머신 배포 마법사는 3-node 클러스터에 UI 서비스 제공을 위한 관리 소프트웨어 스택입니다. 해당 가상머신 역할은 스토리지 시스템 설치와 구성을 하드웨어 설정부터 소프트웨어 설치 및 초기 설정까지 전 과정을 자동화하여 설치 시간을 줄이고, 복잡성을 줄여줍니다. PFMP 가상머신을 배포하기 위해서는 다음의 정보가 필요합니다.

  • 가상머신의 컴퓨트 정보
  • 3-node 네트워크 연결를 위한 NIC 정보
  • SSH Key 설정 배포 정보

필요한 정보를 먼저 준비하십시오. 정보가 준비되었다면 "다음" 버튼을 눌러 가상머신 배포를 시작합니다.

PFMP 가상머신은 관리 소프트웨어로, 설치가 완료된 후에는 더 이상 필요하지 않아 삭제됩니다. 설치 시 최소 권장 사양은 8 vCore의 CPU와 16GiB 메모리로, 이 사양을 기준으로 설치를 진행합니다. 이 후, PFMP 가상머신은 가상화 환경에서 관리 역할을 마치고 시스템 자원을 해제하여 효율성을 높입니다.


Info alert:컴퓨트 자원 구성 시 참고사항

PowerFlex에서 권장되는 최소 CPU는 8 vCore 이상이며, Memory는 8 GiB 이상 입니다.

ROOT Disk의 크기는 100GiB를 디스크가 Thin Provisioning 방식으로 제공됩니다.

PFMP 가상머신을 통하여 스토리지 가상머신을 PowerFlex로 구성하기 위해 NIC를 가상머신에 구성해야 합니다. 가상머신의 NIC를 구성하기 위한 방식을 선택한 후 표시된 장치를 가상머신에 할당합니다.

관리 트래픽 구성
네트워크 구성

Info alert:네트워크 자원 구성 시 참고사항

PFMP 네트워크는 관리 네트워크로 구성됩니다. 따라서 PFMP 가상머신 구성 시 관리 트래픽을 위한 NIC를 할당해야 합니다.

관리 트래픽을 처리하기 위해 가상머신에 NIC를 할당할 때에는 기본적으로 브릿지 방식을 사용합니다. 따라서 해당 호스트에 사용하고자 하는 브릿지가 먼저 생성되어 있어야 합니다. 목록에 표시된 브릿지 중 할당하고자 하는 NIC 브릿지를 선택합니다.

스토리지 네트워크와 같은 대역을 사용해야 하기에 같은 대역의 IP를 입력합니다.

PFMP 가상머신의 설치 소프트웨어 내의 쿠버네티스 컨테이너를 설정하는 네트워크 정보를 입력합니다.

추가 네트워크 구성

Info alert:추가 네트워크 자원 구성 시 참고사항

추가 네트워크에서 요구하는 값은 내부 서비스, 웹 서버, SupportAssist에 사용할 관리 및 모니터링 IP, 스토리지 서버용 IP, 스토리지 복제용 IP입니다.

모든 IP들은 각각 연속적인 5개의 범위가 보장되어야 합니다. 관리 및 모니터링 IP는 스토리지 센터의 관리 대역(연속적인 5개의 범위), 스토리지 서버용 IP는 스토리지 센터의 서버용 IP 대역(연속적인 5개의 범위), 스토리지 복제용 IP는 스토리지 센터의 복제용 IP 대역(연속적인 5개의 범위)으로 설정하셔야 합니다.

스토리지 네트워크와 같은 대역을 사용해야 하기에 같은 대역의 IP를 입력합니다.

호스트 및 스토리지센터 가상머신 간의 암호화된 통신을 위해 생성되는 가상머신에 SSH Key를 설정해야 합니다. 기본적으로 현재 호스트의 SSH Key 파일을 자동으로 등록하며, 필요시 다운로드 한 SSH Key 파일로 등록 가능합니다.


Info alert:SSH Key 등록 참고사항

SSH Key는 호스트 및 스토리지센터 가상머신 등의 ABLESTACK 구성요소 간의 암호화된 인증을 위해 사용됩니다.

호스트 간, 가상머신 간의 모든 명령은 SSH를 이용해 전달되며 이 때 SSH Key를 이용해 인증을 처리합니다. 따라서 모든 호스트, 가상머신은 동일한 SSH Key를 사용해야 합니다.
